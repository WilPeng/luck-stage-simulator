# 赛程状态机设计规范

## 核心概念

整个系统的核心是一个**状态机（State Machine）**。

管理员唯一控制的是：
- **当前是第几公演** (`currentRound`)
- **当前进行到哪个阶段** (`currentStage`)

前端所有页面展示都由这个状态决定，动态计算，不需要存储"已完成"状态。

---

## 一、阶段定义（前后端统一）

### 后端枚举定义

```typescript
enum Stage {
  PREPARATION,     // 预先准备
  CAPTAIN_VOTE,    // 队长选举
  TEAMING,         // 组队
  SONG_SELECT,     // 选歌
  TRAINING,        // 训练
  REHEARSAL,       // 彩排
  PERFORMANCE,     // 公演
  ELIMINATION      // 淘汰
}
```

### 前端类型定义

```typescript
export type StageType =
  | 'preparation'     // 预先准备
  | 'captain_vote'    // 队长选举
  | 'teaming'         // 组队
  | 'song_select'     // 选歌
  | 'training'        // 训练
  | 'rehearsal'       // 彩排
  | 'performance'     // 公演
  | 'elimination'     // 淘汰

// 阶段顺序（固定）
export const STAGE_ORDER: StageType[] = [
  'preparation',
  'captain_vote',
  'teaming',
  'song_select',
  'training',
  'rehearsal',
  'performance',
  'elimination'
]

// 阶段名称映射
export const STAGE_NAMES: Record<StageType, string> = {
  preparation: '预先准备',
  captain_vote: '队长选举',
  teaming: '组队',
  song_select: '选歌',
  training: '训练',
  rehearsal: '彩排',
  performance: '公演',
  elimination: '淘汰'
}
```

### 阶段流转顺序

```
预先准备 → 队长选举 → 组队 → 选歌 → 训练 → 彩排 → 公演 → 淘汰 → 下一公演预先准备
```

```json
    { "round": 1, "stage": "TEAMING", "status": "completed" },
    ...
    { "round": 2, "stage": "TRAINING", "status": "current" },
    { "round": 2, "stage": "REHEARSAL", "status": "future" },
    ...
    { "round": 3, "stage": "PREPARATION", "status": "future" },
    ...
  ]
}
```

### 3.2 管理员设置阶段

```http
POST /admin/progress/set
```

**请求体：**
```json
{
  "round": 2,
  "stage": "TRAINING"
}
```

**响应：** 返回更新后的完整 `matrix`

### 3.3 管理员自动推进下一阶段

```http
POST /admin/progress/next
```

**逻辑：**
- 当前是 `ELIMINATION` → 自动变成 `currentRound + 1, PREPARATION`
- 其他阶段 → 推进到下一个阶段

**示例：**
```
一公淘汰 → 二公预先准备
训练 → 彩排
```

## 四、状态计算规则（核心）

**后端不存储"已完成"状态，全部动态计算。**

### 计算逻辑

给定：
- `currentRound`: 当前轮次
- `currentStage`: 当前阶段
- `targetRound`: 目标轮次
- `targetStage`: 目标阶段

```typescript
function calculateStatus(
  currentRound: number,
  currentStage: Stage,
  targetRound: number,
  targetStage: Stage
): 'completed' | 'current' | 'future' {
  const stageOrder = STAGE_ORDER.indexOf(targetStage)
  const currentStageOrder = STAGE_ORDER.indexOf(currentStage)

  if (targetRound < currentRound) {
    // 过去轮次，全部已完成
    return 'completed'
  }

  if (targetRound > currentRound) {
    // 未来轮次，全部未开始
    return 'future'
  }

  // 当前轮次
  if (stageOrder < currentStageOrder) {
    return 'completed'
  }
  if (stageOrder === currentStageOrder) {
    return 'current'
  }
  return 'future'
}
```

### 示例

当前状态：`{ currentRound: 2, currentStage: "TRAINING" }`

| 轮次 | 阶段 | 计算结果 |
|-----|------|---------|
| 一公 | 所有阶段 | `completed` |
| 二公 | 预先准备 | `completed` |
| 二公 | 队长选举 | `completed` |
| 二公 | 组队 | `completed` |
| 二公 | 选歌 | `completed` |
| 二公 | 训练 | `current` |
| 二公 | 彩排 | `future` |
| 二公 | 公演 | `future` |
| 二公 | 淘汰 | `future` |
| 三公 | 所有阶段 | `future` |

## 八、前端实现要点

### 8.1 统一状态获取

所有页面统一使用：

```typescript
const seasonStore = useSeasonStore()
const status = seasonStore.getStageStatus(round, stage)
```

返回：`'completed' | 'current' | 'future'`

### 8.2 页面统一判断

```vue
<template>
  <div v-if="status === 'current'" class="current-stage">
    <!-- 可操作内容 -->
  </div>
  <div v-else-if="status === 'completed'" class="completed-stage">
    <!-- 只读历史内容 -->
  </div>
  <div v-else class="future-stage">
    <p>该阶段尚未开放</p>
  </div>
</template>

<script setup>
const status = computed(() => seasonStore.getStageStatus(round, stage))
</script>
```

## 九、优势总结

1. **单一状态源**：管理员只控制 `{ currentRound, currentStage }`
2. **动态计算**：不存储"已完成"，避免数据不一致
3. **全局联动**：所有页面自动跟随状态变化
4. **权限清晰**：菜单、按钮、内容全部自动适配
5. **易于维护**：前后端逻辑统一，不会出现"忘记锁定"的问题
