# 后端新增接口清单（按实现优先级排序）

## 优先级 P0：公演结算流程

### 1. 管理员开启公演

```
POST /api/performance/start
```

**权限**: admin  
**请求体**:
```json
{ "round": 1 }
```
**响应**:
```json
{ "success": true, "message": "公演已开启" }
```
**后端逻辑**: 标记该轮次为"公演进行中"，选手端获得操作权限。

---

### 2. 选手生成发挥值

```
POST /api/performance/player-generate
```

**权限**: player（需登录）  
**请求体**:
```json
{ "round": 1 }
```
**响应**:
```json
{
  "success": true,
  "playerId": "u001",
  "performanceValue": 12       // -5 ~ 15 之间的随机整数
}
```
**后端逻辑**: 
- 检查该轮次是否已开启公演
- 生成 -5 ~ 15 的随机整数
- 保存到数据库（每位选手每轮只能生成一次）
- 返回生成的随机值

---

### 3. 管理员代生成

```
POST /api/performance/admin-generate
```

**权限**: admin  
**请求体**:
```json
{ "round": 1, "playerId": "u001" }
```
**响应**: 同接口#2

---

### 4. 管理员一键全生成

```
POST /api/performance/admin-generate-all
```

**权限**: admin  
**请求体**:
```json
{ "round": 1 }
```
**响应**:
```json
{ "success": true, "generatedCount": 33 }
```
**后端逻辑**: 为该轮所有尚未生成的选手生成随机发挥值。

---

### 6. 公演结算（已有接口，新增评级字段）

```
POST /api/performance/calculate
```

**请求体**:
```json
{ "roundId": "round-1" }
```
**响应（新增字段）**:

`TeamPerformanceResult` 新增字段：
```json
{
  "teamScore": 82,              // 团队综合得分
  "teamRating": "A",            // S/A/B/C/D
  "teamRatingText": "出色表现"
}
```

`PlayerPerformanceResult` 新增字段：
```json
{
  "playerScore": 78,            // 选手个人得分 0~120
  "stageRating": "A",           // S/A/B/C/D
  "stageRatingText": "出色表现"
}
```

**评级计算规则**：

```
attributeScore = player.vocal × normalizedVocalW
               + player.dance × normalizedDanceW
               + player.charm × normalizedCharmW

difficultyFactor = 1 - (song.difficulty - 1) × 0.1
performanceBonus = (选手端生成的随机值) × 2

playerFinalScore = attributeScore × difficultyFactor + performanceBonus

评级映射：
  S(完美舞台) ≥ 85
  A(出色表现) ≥ 65
  B(稳定发挥) ≥ 45
  C(略有不足) ≥ 25
  D(失误较多) < 25

teamScore = avg(所有成员的 playerFinalScore) + random(0, 20)
团队评级使用相同映射
```

### 7. 释放个人喜爱度最终结果

```
POST /api/admin/audience-vote/release
```

**权限**: admin  
**请求体**:
```json
{
  "round": 1,
  "rankings": [
    { "playerId": "u001", "playerName": "张三", "teamId": "round-1-team-1", "teamName": "星光队", "votes": 892, "rank": 1, "totalWeight": 118 },
    { "playerId": "u002", "playerName": "李四", "teamId": "round-1-team-1", "teamName": "星光队", "votes": 760, "rank": 2, "totalWeight": 95 }
  ]
}
```
**响应**:
```json
{ "success": true, "message": "个人喜爱度结果已释放" }
```
**后端逻辑**: 将最终排名保存到数据库，之后淘汰阶段可以读取。

### 8. 获取最终个人喜爱度排名

```
GET /api/audience-vote/final-ranking?round=1
```
**响应**:
```json
{
  "success": true,
  "released": true,
  "rankings": [
    { "playerId": "u001", "playerName": "张三", "teamId": "round-1-team-1", "teamName": "星光队", "votes": 892, "rank": 1 }
  ]
}
```

## 优先级 P1：大众评审微调

### 9. 微调某个评审的投票

```
PUT /api/admin/audience-vote/seat/{seatNumber}
```

**权限**: admin  
**请求体**:
```json
{
  "round": 1,
  "votes": [
    { "voteOrder": 1, "playerId": "u001", "playerName": "张三" },
    { "voteOrder": 2, "playerId": "u004", "playerName": "赵六" },
    { "voteOrder": 3, "playerId": "u007", "playerName": "周九" }
  ]
}
```
**响应**:
```json
{ "success": true }
```

## 接口汇总

| # | 优先级 | 方法 | 端点 | 说明 |
|---|--------|------|------|------|
| 1 | P0 | POST | `/api/performance/start` | 管理员开启公演 |
| 2 | P0 | POST | `/api/performance/player-generate` | 选手生成发挥值 |
| 3 | P0 | POST | `/api/performance/admin-generate` | 管理员代生成 |
| 4 | P0 | POST | `/api/performance/admin-generate-all` | 一键全生成 |
| 5 | P0 | GET | `/api/performance/player-status?round=` | 查询选手状态 |
| 6 | P0 | POST | `/api/performance/calculate` | **已有**，新增评级字段 |
| 7 | P0 | POST | `/api/admin/audience-vote/release` | 释放最终排名 |
| 8 | P0 | GET | `/api/audience-vote/final-ranking?round=` | 获取最终排名 |
| 9 | P1 | PUT | `/api/admin/audience-vote/seat/:num` | 微调评审投票 |
