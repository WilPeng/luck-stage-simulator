# 公演结算模块 - 后端接口规范

**更新日期**: 2026-06-23  
**对应前端版本**: 已完成修复（currentRoundId 获取逻辑、API 调用方式）

---

## 通用说明

| 项目 | 要求 |
|------|------|
| 服务地址 | `http://localhost:3000` |
| API 前缀 | `/api` |
| 认证方式 | Header: `Authorization: Bearer <token>` |
| 请求头 | `Content-Type: application/json` |
| 成功响应格式 | `{ "success": true, "data": <数据> }` 或 `{ "data": <数据> }` |
| 失败响应格式 | `{ "success": false, "message": "错误信息" }` 或 HTTP 状态码 + `{ "error": "..." }` |

> **关键**：前端 `doRequest` 会优先取响应中的 `json.data` 字段返回。所以后端响应建议包在 `data` 字段里。

## 一、核心公演结算接口

### 1.1 执行公演结算（新结构）

- **方法**: `POST`
- **URL**: `/api/performance/calculate`
- **权限**: 管理员
- **请求体**:
  ```json
  {
    "roundId": "round_1"   // 必填，轮次 ID
  }
  ```
- **响应格式**:
  ```json
  {
    "success": true,
    "data": {
      "teamResults": [
        {
          "teamId": "team_1",
          "teamName": "队伍名",
          "songName": "歌曲名",
          "memberCount": 5,
          "finalVotes": 1200,
          "rank": 1,
          "status": "calculated",
          "songWeights": { "vocal": 0.3, "dance": 0.4, "charm": 0.3 },
          "memberResults": [...]
        }
      ],
      "playerResults": [
        {
          "playerId": "u001",
          "playerName": "选手名",
          "teamId": "team_1",
          "teamName": "队伍名",
          "finalVotes": 300,
          "contribution": 150,
          "rankInTeam": 1,
          "rankOverall": 1
        }
      ]
    }
  }
  ```

### 1.2 获取公演结算结果（管理员）

- **方法**: `GET`
- **URL**: `/api/performance/result?roundId=round_1`
- **权限**: 管理员
- **查询参数**: `roundId`（必填）
- **响应格式**: 同 1.1

### 1.3 获取公演结算结果（选手端）

- **方法**: `GET`
- **URL**: `/api/player/performance/result?roundId=round_1`
- **权限**: 需要登录
- **查询参数**: `roundId`（必填）
- **响应格式**: 同 1.1（但可能只返回当前选手所在队伍的信息）

- **方法**: `POST`
- **URL**: `/api/performance/calculate`
- **权限**: 管理员
- **请求体**: `{ "round": 1 }`

## 三、大众评审喜爱度投票接口

- **方法**: `POST`

- **请求体**: `{ "roundId": "round_1" }`

## 五、前端修复说明

### 已修复的问题

1. **`currentRoundId` 计算逻辑增强**:
   - 原逻辑：依赖 `seasonStore.rounds` 列表（需要 `GET /api/season/rounds`）
   - 修复后：增加了 fallback 链，即使 `rounds` API 404，也能通过其他方式获取 `roundId`

2. **API 调用方式修改**:
   - 原方式：使用自定义 Header `X-Round-Id` 传递 `roundId`
   - 修复后：改为 URL 查询参数 `?roundId=xxx`
   - 影响接口：
     - `GET /api/performance/result?roundId=xxx`
     - `GET /api/player/performance/result?roundId=xxx`

### 前端当前行为

1. 页面加载时，会调用 `seasonStore.fetchSeason()` 和 `seasonStore.fetchRounds()`
2. 如果 `fetchRounds()` 失败（404），前端会使用 fallback 逻辑构造一个 `roundId`（格式：`round_<轮次编号>`）
3. 然后使用这个 `roundId` 调用 `fetchAdminPerformanceResults(roundId)` 获取公演结果
