## 产品概述

修复选秀模拟系统中轮次数据隔离问题，确保队长选举、组队、选歌的结果按轮次独立存储，不会因页面切换或进入公演而丢失。

## 核心功能

1. 按轮次隔离存储队伍数据（RoundTeam）
2. 按轮次隔离存储歌曲分配数据（TeamSong）
3. 确保 Mock 模式下数据持久化到 localStorage
4. 修复选歌页面显示"未知队伍"的问题
5. 确保同轮次内各环节数据正常共享

## 技术栈

- 前端框架：Vue 3 + TypeScript
- 状态管理：Pinia
- 持久化存储：localStorage
- 现有模块：dataService.ts（已创建，提供按轮次隔离数据的底层能力）

## 实现方案

### 问题分析

1. **API 层 Mock 实现返回空数组**：`api.ts` 中的 `setupRoundTeams`、`randomAssignTeams`、`manualAssignTeams`、`assignTeamSongs` 等方法的 Mock 实现返回空数组，导致数据无法持久化
2. **Store 层未集成 dataService**：`teamStore.ts` 和 `songStore.ts` 直接使用 API，未在关键操作后保存数据到 dataService
3. **选歌页面"未知队伍"**：`AdminSongView.vue` 中的 `getTeamName` 方法查找 `teams` 时，可能因为数据未加载或 ID 格式不匹配而返回"未知队伍"

### 修改方案

#### 1. 修改 `api.ts`，集成 dataService

- 修改 `setupRoundTeams` 的 Mock 实现，保存数据到 dataService
- 修改 `randomAssignTeams` 的 Mock 实现，保存数据到 dataService
- 修改 `manualAssignTeams` 的 Mock 实现，保存数据到 dataService
- 修改 `assignTeamSongs` 的 Mock 实现，保存数据到 dataService
- 修改 `setTeamCaptain` 的 Mock 实现，保存数据到 dataService
- 确保 `getRoundTeams` 和 `getTeamSongs` 的 Mock 实现从 dataService 读取数据

#### 2. 修改 `teamStore.ts`，在关键操作后保存数据

- 在 `fetchTeams` 成功后，从 dataService 读取数据并更新 store
- 在 `assignCaptain` 成功后，保存数据到 dataService
- 在 `addMember` 成功后，保存数据到 dataService
- 在 `removeMember` 成功后，保存数据到 dataService
- 在 `lockTeam` 成功后，保存数据到 dataService
- 在 `unlockTeam` 成功后，保存数据到 dataService

#### 3. 修改 `songStore.ts`，在关键操作后保存数据

- 在 `fetchTeamSongs` 成功后，从 dataService 读取数据并更新 store
- 在 `assignTeamSongs` 成功后，保存数据到 dataService
- 在 `releaseSong` 成功后，保存数据到 dataService
- 在 `claimSong` 成功后，保存数据到 dataService

#### 4. 修复 `AdminSongView.vue` 的"未知队伍"问题

- 确保 `refreshData` 正确加载 teams 数据
- 修改 `getTeamName` 方法，添加调试日志
- 确保 `teams` 响应式数据在 `refreshData` 后正确更新

#### 5. 确保数据持久化

- 所有关键操作后，数据同时保存到 dataService（localStorage）
- 页面加载时，优先从 dataService 读取数据
- 如果 dataService 中没有数据，再调用 API

## 架构设计

### 数据流

```
用户操作 → Store 方法 → API 层 → Mock 实现 → dataService → localStorage
                     ↓
                 真实后端 API → 数据库
```

### 数据存储格式

- 队伍数据：`luck_sim_round_{roundId}_teams`
- 歌曲分配数据：`luck_sim_round_{roundId}_teamSongs`
- 按轮次隔离，不同轮次的数据互不影响

### 2. Store 层保存数据

修改 `teamStore.ts`，在关键操作后保存数据：

```typescript
async function assignCaptain(teamId: string, playerId: string, roundId?: string): Promise<RoundTeam> {
  const rid = roundId || currentRoundId.value
  if (!rid) {
    throw new Error('No roundId provided')
  }

  try {
    const result = await apiSetTeamCaptain(teamId, playerId)
    
    // 保存到 dataService
    const teams = teams.value.map(t => t.id === result.id ? result : t)
    saveTeams(rid, teams)
    
    await fetchTeams(rid)
    return result
  } catch (e) {
    console.error('[TeamStore] assignCaptain error:', e)
    throw e
  }
}
```

### 3. 修复"未知队伍"问题

修改 `AdminSongView.vue`：

```typescript
function getTeamName(teamId: string): string {
  // 确保 teams 已加载
  if (teams.value.length === 0) {
    console.warn('[AdminSongView] teams 为空，无法获取队伍名称')
    return '未知队伍'
  }
  
  const team = teams.value.find(t => t.id === teamId)
  if (!team) {
    console.warn(`[AdminSongView] 未找到队伍: ${teamId}`, teams.value)
    return '未知队伍'
  }
  return team.name || '未知队伍'
}
```

### 4. 确保数据加载顺序

修改 `AdminSongView.vue` 的 `refreshData`：

```typescript
async function refreshData() {
  loading.value = true
  try {
    const roundId = seasonStore.currentRoundId
    if (!roundId) {
      MessagePlugin.error('当前轮次未设置')
      return
    }
    
    // 先加载队伍数据，再加载歌曲数据
    await teamStore.fetchTeams(roundId)
    await Promise.all([
      songStore.fetchRoundSongs(roundId),
      songStore.fetchTeamSongs(roundId),
      songStore.fetchSongs()
    ])
  } catch (error: any) {
    MessagePlugin.error(error.message || '刷新失败')
  } finally {
    loading.value = false
  }
}
```
