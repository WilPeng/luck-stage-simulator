# 02-后端开发规范

> **本文档引用《01-核心业务模型》，后端开发必须遵循本文档定义的所有接口和数据结构。**

---

## 一、基础规范

### 1.1 API 路径规范

- 所有接口必须以 `/api` 开头
- 管理员接口以 `/api/admin` 开头
- 用户接口以 `/api/player` 开头
- 公开接口以 `/api/public` 开头

### 1.2 HTTP 方法规范

| 方法 | 用途 |
|------|------|
| GET | 查询/获取数据 |
| POST | 创建资源 |
| PUT | 更新资源 |
| DELETE | 删除资源 |
| PATCH | 部分更新 |

### 1.3 响应格式规范

```typescript
// 成功响应
interface SuccessResponse<T> {
  success: true
  data: T
}

// 错误响应
interface ErrorResponse {
  success: false
  error: {
    code: string
    message: string
    details?: any
  }
}
```

### 1.4 分页响应格式

```typescript
interface PaginatedResponse<T> {
  success: true
  data: {
    list: T[]
    total: number
    page: number
    pageSize: number
    totalPages: number
  }
}
```

## 二、阶段控制系统接口

### 2.1 获取当前赛程进度（核心接口）

```http
GET /api/season/progress
```

**响应：**

```json
{
  "success": true,
  "data": {
    "currentRound": 1,
    "currentStage": "TEAMING"
  }
}
```

**说明：** 这是阶段控制系统的核心接口，所有前端页面依赖此接口获取当前状态。

---

### 2.2 获取状态矩阵

```http
GET /api/season/progress/matrix
```

**响应：**

```json
{
  "success": true,
  "data": {
    "currentRound": 2,
    "currentStage": "TRAINING",
    "matrix": [
      { "round": 1, "stage": "PREPARATION", "status": "completed" },
      { "round": 1, "stage": "CAPTAIN_VOTE", "status": "completed" },
      { "round": 1, "stage": "TEAMING", "status": "completed" },
      { "round": 1, "stage": "SONG_SELECT", "status": "completed" },
      { "round": 1, "stage": "TRAINING", "status": "completed" },
      { "round": 1, "stage": "REHEARSAL", "status": "completed" },
      { "round": 1, "stage": "PERFORMANCE", "status": "completed" },
      { "round": 1, "stage": "ELIMINATION", "status": "completed" },
      { "round": 2, "stage": "PREPARATION", "status": "completed" },
      { "round": 2, "stage": "CAPTAIN_VOTE", "status": "completed" },
      { "round": 2, "stage": "TEAMING", "status": "completed" },
      { "round": 2, "stage": "SONG_SELECT", "status": "completed" },
      { "round": 2, "stage": "TRAINING", "status": "current" },
      { "round": 2, "stage": "REHEARSAL", "status": "future" },
      { "round": 2, "stage": "PERFORMANCE", "status": "future" },
      { "round": 2, "stage": "ELIMINATION", "status": "future" },
      { "round": 3, "stage": "PREPARATION", "status": "future" }
      // ... 继续到 totalRounds
    ]
  }
}
```

**status 说明：**

| 值 | 含义 |
|-----|------|
| `completed` | 已完成 |
| `current` | 当前阶段 |
| `future` | 未开始 |

**⚠️ 重要：状态必须由后端计算返回，前端禁止自行计算。**

---

### 2.3 设置赛程状态（管理员操作）

```http
POST /api/admin/progress/set
```

**请求体：**

```json
{
  "round": 2,
  "stage": "TRAINING"
}
```

**响应：**

```json
{
  "success": true,
  "data": {
    "currentRound": 2,
    "currentStage": "TRAINING"
  }
}
```

**验证规则：**

1. `round` 必须在 1 到 `totalRounds` 之间
2. `stage` 必须是有效的 Stage 值
3. 不能跳阶段设置（只能设置当前阶段的下一个阶段）

---

### 2.4 自动推进下一阶段（管理员操作）

```http
POST /api/admin/progress/next
```

**逻辑：**

```
当前状态：
  { currentRound: 1, currentStage: "TEAMING" }
  
推进后：
  { currentRound: 1, currentStage: "SONG_SELECT" }
```

**特殊情况：**

```
当前状态：
  { currentRound: 1, currentStage: "ELIMINATION" }
  
推进后：
  { currentRound: 2, currentStage: "PREPARATION" }
```

**响应：**

```json
{
  "success": true,
  "data": {
    "previousRound": 1,
    "previousStage": "ELIMINATION",
    "currentRound": 2,
    "currentStage": "PREPARATION"
  }
}
```

## 十三、错误码规范

```typescript
const ErrorCodes = {
  // 认证错误 1000-1999
  AUTH_TOKEN_EXPIRED: 'AUTH_1001',
  AUTH_TOKEN_INVALID: 'AUTH_1002',

  // 参数错误 2000-2999

}
```
