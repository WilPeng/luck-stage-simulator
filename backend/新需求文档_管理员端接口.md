# 管理员端核心功能接口 - 新需求文档

## 一、接口概述

本文档为前端管理员端页面提供所需的后端接口需求，包含以下模块：
1. 公演轮次配置接口
2. 训练统计接口
3. 淘汰管理接口（扩展）

---

## 二、公演轮次配置接口

### 2.1 获取公演轮次配置

**接口路径**: `/api/season/performance-round`
**请求方法**: `GET`
**功能**: 获取当前公演轮次配置信息

#### 请求
```
GET /api/season/performance-round
Headers: { Authorization: Bearer <token> }
```

#### 响应
```json
{
  "success": true,
  "data": {
    "id": "pr001",
    "name": "乘风2026第一轮公演",
    "status": "pending" | "active" | "finished",
    "config": {
      "teamCount": 4,
      "maxMembersPerTeam": 6,
      "songPoolIds": ["s001", "s002", "s003", "s004", "s005", "s006"],
      "trainingTimesAllowed": 5,
      "eliminationCount": 2,
      "dangerLineRatio": 0.5
    },
    "createdAt": "2026-06-10T10:00:00Z",
    "startedAt": null,
    "finishedAt": null
  }
}
```

#### 字段说明
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 轮次ID |
| name | string | 轮次名称，如"第一轮公演" |
| status | enum | `pending` 待开始 / `active` 进行中 / `finished` 已结束 |
| config.teamCount | number | 参与队伍数量 |
| config.maxMembersPerTeam | number | 每队最大人数 |
| config.songPoolIds | string[] | 可选歌曲ID列表 |
| config.trainingTimesAllowed | number | 允许的训练次数 |
| config.eliminationCount | number | 淘汰人数 |
| config.dangerLineRatio | number | 危险线比例（0.5表示后50%为危险队伍） |
| createdAt | string | 创建时间 |
| startedAt | string | 开始时间 |
| finishedAt | string | 结束时间 |

---

### 2.2 创建公演轮次

**接口路径**: `/api/season/performance-round/create`
**请求方法**: `POST`
**功能**: 管理员创建新的公演轮次

#### 请求
```json
{
  "name": "乘风2026第一轮公演",
  "config": {
    "teamCount": 4,
    "maxMembersPerTeam": 6,
    "songPoolIds": ["s001", "s002", "s003", "s004", "s005", "s006"],
    "trainingTimesAllowed": 5,
    "eliminationCount": 2,
    "dangerLineRatio": 0.5
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 轮次名称 |
| config.teamCount | number | 是 | 参与队伍数量 |
| config.maxMembersPerTeam | number | 是 | 每队最大人数 |
| config.songPoolIds | string[] | 是 | 可选歌曲ID列表 |
| config.trainingTimesAllowed | number | 否 | 默认5次 |
| config.eliminationCount | number | 是 | 淘汰人数 |
| config.dangerLineRatio | number | 否 | 默认0.5 |

#### 响应（成功）
```json
{
  "success": true,
  "data": {
    "id": "pr001",
    "name": "乘风2026第一轮公演",
    "status": "pending",
    "config": { ... },
    "createdAt": "2026-06-10T10:00:00Z"
  }
}
```

#### 业务规则
- 只能有一个 `active` 状态的轮次
- 新创建的轮次 status 为 `pending`
- 创建后自动将赛季 currentStage 更新为 `training`

## 三、训练统计接口

### 3.1 获取训练统计数据

**接口路径**: `/api/training/stats`
**请求方法**: `GET`
**功能**: 获取所有选手的训练进度统计

#### 请求
```
GET /api/training/stats
Headers: { Authorization: Bearer <token> }
```

#### 响应
```json
{
  "success": true,
  "data": {
    "totalPlayers": 16,
    "activePlayers": 14,
    "playersInTeams": 12,
    "playersNotInTeams": 4,
    "totalTrainingCount": 45,
    "averageTrainingCount": 3.2,
    "completedTrainingPlayers": 10,
    "completionRate": 0.625,
    "trainingDistribution": {
      "0次": 4,
      "1-2次": 2,
      "3-4次": 6,
      "5次": 4
    },
    "topImprovers": [
      { "userId": "u001", "name": "选手A", "improvement": 15 },
      { "userId": "u002", "name": "选手B", "improvement": 12 }
    ]
  }
}
```

#### 字段说明
| 字段 | 类型 | 说明 |
|------|------|------|
| totalPlayers | number | 选手总数 |
| activePlayers | number | 活跃选手数（未淘汰） |
| playersInTeams | number | 已组队人数 |
| playersNotInTeams | number | 未组队人数 |
| totalTrainingCount | number | 总训练次数 |
| averageTrainingCount | number | 人均训练次数 |
| completedTrainingPlayers | number | 完成训练人数（达到上限） |
| completionRate | number | 完成率 |
| trainingDistribution | object | 训练次数分布 |
| topImprovers | array | 属性提升最多的选手 |

## 四、淘汰管理接口（扩展）

### 4.1 获取选手排名列表

**接口路径**: `/api/elimination/ranking`
**请求方法**: `GET`
**功能**: 获取所有选手的排名列表（含状态）

#### 请求
```
GET /api/elimination/ranking
Headers: { Authorization: Bearer <token> }
```

#### 响应
```json
{
  "success": true,
  "data": {
    "roundId": "pr001",
    "rankings": [
      {
        "rank": 1,
        "userId": "u001",
        "name": "选手A",
        "teamId": "t001",
        "teamName": "星光队",
        "finalScore": 285,
        "status": "safe",
        "isDanger": false
      },
      {
        "rank": 8,
        "userId": "u008",
        "name": "选手H",
        "teamId": "t002",
        "teamName": "梦想队",
        "finalScore": 245,
        "status": "danger",
        "isDanger": true
      },
      {
        "rank": 12,
        "userId": "u012",
        "name": "选手L",
        "teamId": null,
        "teamName": null,
        "finalScore": null,
        "status": "eliminated",
        "isDanger": false
      }
    ],
    "dangerLine": 8,
    "dangerTeamIds": ["t002", "t003"]
  }
}
```

#### 字段说明
| 字段 | 类型 | 说明 |
|------|------|------|
| roundId | string | 当前轮次ID |
| rankings | array | 排名列表 |
| rankings[].rank | number | 排名 |
| rankings[].userId | string | 用户ID |
| rankings[].name | string | 用户姓名 |
| rankings[].teamId | string | 队伍ID |
| rankings[].teamName | string | 队伍名称 |
| rankings[].finalScore | number | 最终得分（无得分时为null） |
| rankings[].status | enum | `safe` 安全 / `danger` 危险 / `eliminated` 已淘汰 |
| rankings[].isDanger | boolean | 是否处于危险区 |
| dangerLine | number | 危险线排名 |
| dangerTeamIds | string[] | 危险队伍ID列表 |

### 4.4 执行淘汰

**接口路径**: `/api/elimination/execute`
**请求方法**: `POST`
**功能**: 正式执行淘汰操作

#### 请求
```
POST /api/elimination/execute
Headers: { Authorization: Bearer <token> }
Body: { "roundId": "pr001" }
```

#### 响应
```json
{
  "success": true,
  "data": {
    "roundId": "pr001",
    "eliminatedUsers": [
      { "userId": "u008", "name": "选手H" },
      { "userId": "u011", "name": "选手K" }
    ],
    "executedAt": "2026-06-10T15:30:00Z"
  }
}
```

#### 业务规则
- 执行后更新用户 status = 'eliminated'，teamId = null
- 更新轮次状态为 finished
- 记录操作日志

---

### 4.5 手动淘汰

**接口路径**: `/api/elimination/manual`
**请求方法**: `POST`
**功能**: 管理员手动指定淘汰某个选手

#### 请求
```json
{
  "userId": "u005",
  "reason": "管理员手动淘汰"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | string | 是 | 要淘汰的用户ID |
| reason | string | 是 | 淘汰原因 |

#### 响应
```json
{
  "success": true,
  "data": {
    "userId": "u005",
    "name": "选手E",
    "eliminatedAt": "2026-06-10T16:00:00Z"
  }
}
```

#### 业务规则
- 仅管理员可操作
- 淘汰后用户 status = 'eliminated'，teamId = null
- 记录操作日志，包含手动淘汰原因

---

### 4.6 恢复选手

**接口路径**: `/api/elimination/restore/:userId`
**请求方法**: `POST`
**功能**: 恢复已淘汰的选手

#### 请求
```
POST /api/elimination/restore/u005
Headers: { Authorization: Bearer <token> }
Body: { "reason": "误淘汰恢复" }
```

#### 响应
```json
{
  "success": true,
  "data": {
    "userId": "u005",
    "name": "选手E",
    "status": "active",
    "restoredAt": "2026-06-10T16:30:00Z"
  }
}
```

#### 业务规则
- 仅管理员可操作
- 恢复后用户 status = 'active'
- 不自动恢复 teamId（需手动分配队伍）
- 记录操作日志

## 七、错误响应格式

```json
{
  "success": false,
  "error": "错误描述",
  "code": "ERROR_CODE"
}
```

### 错误码说明
| code | 说明 |
|------|------|
| NO_TOKEN | 未提供认证令牌 |
| INVALID_TOKEN | 令牌无效或过期 |
| FORBIDDEN | 权限不足 |
| STAGE_NOT_ALLOWED | 当前阶段不允许此操作 |
| ROUND_NOT_FOUND | 轮次不存在 |
| ROUND_ALREADY_ACTIVE | 已有进行中的轮次 |
| USER_NOT_FOUND | 用户不存在 |
| USER_ALREADY_ELIMINATED | 用户已被淘汰 |
| ELIMINATION_ALREADY_EXECUTED | 本轮淘汰已执行 |
