## 产品概述

实现公演评级体系后端对齐文档要求，按照新公式计算选手个人得分、评级、团队得分、团队评级和最终票数。

## 核心功能

1. 选手个人得分计算：attributeScore × difficultyFactor + performanceBonus
2. 选手评级映射：S(≥85)/A(≥65)/B(≥45)/C(≥25)/D(<25)
3. 团队得分计算：avg(成员个人分) + random(0,20)
4. 团队评级映射：同选手阈值
5. 最终票数计算：500 + teamScore × 3 + random(-10,20)
6. 在 Model 和接口返回中新增评级相关字段

## 技术选型

- 基于现有 Node.js + Express 后端架构
- 复用现有 PerformanceValue、PlayerPerformance、TeamPerformance、Song 等 Model
- 复用 randomInt 工具函数

## 实现方案

### 1. Model 新增字段

**PlayerPerformance.js** 新增：

- `playerScore` (Number) — 选手个人得分 0~120
- `stageRating` (String) — S/A/B/C/D
- `stageRatingText` (String) — 评级描述

**TeamPerformance.js** 新增：

- `teamScore` (Number) — 团队综合得分
- `teamRating` (String) — S/A/B/C/D
- `teamRatingText` (String) — 评级描述

### 2. 重写 calculate 端点

核心计算逻辑：

1. 从 `PerformanceValue` 表读取选手发挥值
2. 若部分选手未生成发挥值，自动补生成
3. 对每个选手计算：

- `normalizedVocalW = song.vocalWeight / totalWeight`
- `normalizedDanceW = song.danceWeight / totalWeight`
- `normalizedCharmW = song.charmWeight / totalWeight`
- `attributeScore = player.vocal × normalizedVocalW + player.dance × normalizedDanceW + player.charm × normalizedCharmW`
- `difficultyFactor = 1 - (song.difficulty - 1) × 0.1`
- `performanceBonus = performanceValue × 2`
- `playerFinalScore = Math.round(attributeScore × difficultyFactor + performanceBonus)`
- `stageRating` = 根据 playerFinalScore 映射

4. 对每个队伍计算：

- `teamAvgScore = round(average of member's playerFinalScore)`
- `teamBonus = randomInt(0, 20)`
- `teamScore = teamAvgScore + teamBonus`
- `teamRating` = 根据 teamScore 映射

5. 计算 `finalVotes = 500 + teamScore × 3 + randomInt(-10, 20)`

### 3. 调整 reveal-team 端点

使用与 calculate 相同的评级计算逻辑，返回新增字段。

## 架构设计

performance.js 中的 calculate 和 reveal-team 端点共享相同的评级计算函数，避免逻辑重复。

提取公共函数：

- `calcPlayerScore(player, song)` — 计算机个人分和评级
- `calcTeamScore(members)` — 计算团队分和团队评级
- `getRating(score)` — 根据分数返回评级和描述
