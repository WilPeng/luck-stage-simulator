# 公演数据持久化 — 后端接口文档

## 概述

公演页面（选手发挥 → 结算 → 喜爱度票数）各步骤的数据需要持久化存储，
刷新页面后能从后端恢复，不需要从头开始。

## 新增接口

### 1. 保存选手发挥值

```
POST /api/performance/player-status/save
权限: admin
Body: {
  "roundId": "round-1",
  "players": [
    { "playerId": "u001", "performanceValue": 12 },
    { "playerId": "u002", "performanceValue": -3 }
  ]
}
响应: { "success": true }
```
说明：保存本轮所有选手的发挥值（-5~15）。覆盖式保存，同选手重复调用以最后一次为准。

### 2. 获取选手发挥值

```
GET /api/performance/player-status?roundId=round-1
响应: {
  "success": true,
  "started": true,                  // 公演是否已开启
  "players": [
    { "playerId": "u001", "playerName": "张三", "teamId": "round-1-team-1", "teamName": "星光队", "generated": true, "performanceValue": 12 },
    { "playerId": "u002", "playerName": "李四", "teamId": "round-1-team-1", "teamName": "星光队", "generated": false, "performanceValue": null }
  ]
}
```
说明：获取本轮所有选手的发挥状态。`started` 表示管理员是否已点击"选手开始公演"。

### 3. 保存公演开启状态

```
POST /api/performance/start
权限: admin
Body: { "roundId": "round-1" }
响应: { "success": true, "message": "公演已开启" }
```
说明：标记本轮公演已开启，选手端获得操作权限。等价于"选手开始公演"按钮。

### 4. 保存已揭晓队伍

```
POST /api/performance/revealed-teams/save
权限: admin
Body: {
  "roundId": "round-1",
  "revealedTeamIds": ["round-1-team-1", "round-1-team-2"]
}
响应: { "success": true }
```

### 5. 获取已揭晓队伍

```
GET /api/performance/revealed-teams?roundId=round-1
响应: {
  "success": true,
  "revealedTeamIds": ["round-1-team-1", "round-1-team-2"],
  "allRevealed": false
}
```

## 前端恢复逻辑

页面刷新时，前端按以下顺序恢复：

```
onMounted
  → GET /api/performance/player-status?roundId=round-1
    → 有数据：恢复选手列表和公演开启状态
  → GET /api/performance/revealed-teams?roundId=round-1
    → 有数据：恢复已揭晓队伍，可能跳到阶段二
  → 检查 performanceStore.teamPerformanceResults
    → 有数据：说明已结算，激活阶段二
  → 检查 audienceVoteGenerated
    → true：激活阶段三（喜爱度票数）
```

## 不需要额外接口的持久化

以下数据已通过现有接口持久化：

| 数据 | 已有接口 |
|------|---------|
| 公演结算结果 | `POST /api/performance/calculate` |
| 观众投票数据 | `POST /api/admin/audience-vote/generate` |
| 释放结果 | `POST /api/admin/audience-vote/release` |
| 微调投票 | `PUT /api/admin/audience-vote/seat/:num` |
