# 阶段控制系统需求说明

## 第一部分：产品目标

整个系统所有页面都由一个唯一状态驱动：

```typescript
{
  currentRound: number,    // 当前第几公演（1, 2, 3...）
  currentStage: Stage      // 当前阶段
}
```

例如：

```json
{
  "currentRound": 1,
  "currentStage": "TEAMING"
}
```

表示：**当前正在进行第一公演组队阶段**

所有页面权限都根据这个状态自动计算。

## 第二部分：阶段定义（前后端必须完全一致）

### 后端枚举定义

```typescript
export enum Stage {
  PREPARATION,     // 预先准备
  CAPTAIN_VOTE,    // 队长选举
  TEAMING,         // 组队
  SONG_SELECT,     // 选歌
  TRAINING,        // 训练
  REHEARSAL,       // 彩排
  PERFORMANCE,     // 公演
  ELIMINATION      // 淘汰
}
```

### 前端类型定义

```typescript
export type StageType =
  | 'preparation'     // 预先准备
  | 'captain_vote'    // 队长选举
  | 'teaming'         // 组队
  | 'song_select'     // 选歌
  | 'training'        // 训练
  | 'rehearsal'       // 彩排
  | 'performance'     // 公演
  | 'elimination'     // 淘汰
```

### 阶段顺序（固定）

```
PREPARATION → CAPTAIN_VOTE → TEAMING → SONG_SELECT → TRAINING → REHEARSAL → PERFORMANCE → ELIMINATION
```

**任何地方不得自己写字符串**，全部使用统一枚举。

## 第四部分：状态计算规则（后端实现）

**后端不存储"已完成"状态，全部动态计算。**

```typescript
function calculateStatus(
  currentRound: number,
  currentStage: Stage,
  targetRound: number,
  targetStage: Stage
): 'completed' | 'current' | 'future' {

  const stageOrder = [
    'PREPARATION', 'CAPTAIN_VOTE', 'TEAMING', 'SONG_SELECT',
    'TRAINING', 'REHEARSAL', 'PERFORMANCE', 'ELIMINATION'
  ]

  const currentStageIndex = stageOrder.indexOf(currentStage)
  const targetStageIndex = stageOrder.indexOf(targetStage)

  if (targetRound < currentRound) {
    return 'completed'
  }

  if (targetRound > currentRound) {
    return 'future'
  }

  // 当前轮次
  if (targetStageIndex < currentStageIndex) {
    return 'completed'
  }
  if (targetStageIndex === currentStageIndex) {
    return 'current'
  }
  return 'future'
}
```

## 第五部分：前端实现逻辑

### 管理员页面

展示矩阵：

| 阶段 | 一公 | 二公 | 三公 |
|-----|------|------|------|
| 预先准备 | ✓ | 当前 | |
| 队长选举 | ✓ | | |
| 组队 | ✓ | | |

点击任意格，发送 `POST /api/admin/progress/set`，成功后重新调用 `GET /api/season/progress/matrix` 刷新界面。

### 用户菜单逻辑

菜单结构：

```
首页
聊天

第一公演
    队长选举
    组队
    选歌
    训练
    彩排
    公演
    淘汰

第二公演
...
```

每个菜单项对应：

```json
{
  "round": 1,
  "stage": "TEAMING",
  "status": "completed"
}
```

**规则：**

| 状态 | 可进入 | 模式 |
|------|--------|------|
| `completed` | ✓ | 只读 |
| `current` | ✓ | 可操作 |
| `future` | ✗ | 禁用 |

## 第六部分：所有业务页面统一规则

例如组队页面：

| 状态 | 展示内容 |
|------|---------|
| `current` | 邀请、申请、接受、拒绝按钮 |
| `completed` | 最终组队结果（只读），隐藏所有按钮 |
| `future` | "该阶段尚未开放"提示 |
