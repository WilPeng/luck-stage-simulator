# 公演结算完整流程设计

## 整体流程

```
管理员进入公演页
  → 展示概览（队伍数/选手数/歌曲数）
  → 点击"选手开始公演"（给选手端放权）
  → 选手端各自生成随机发挥值
  → 管理员表格实时查看进度，也可代为生成
  → 全员就绪 → 管理员点击"开始结算"
  → 逐一揭晓队伍（显示歌曲、成员、个人评级、团队得分）
  → 全部揭晓 → 进入"喜爱度票数"标签页
  → 编辑选手权重 → 生成大众评审投票
  → 微调评审投票 → 点击"释放"确定最终结果
  → 进入淘汰阶段
```

## 后端接口清单

### 阶段一：选手发挥

| # | 方法 | 端点 | 说明 | 权限 |
|---|------|------|------|------|
| 1 | POST | `/api/performance/start` | 管理员开启本轮公演，给选手端放权 | admin |
| 2 | POST | `/api/performance/player-generate` | 选手端生成随机发挥值 | player |
| 3 | POST | `/api/performance/admin-generate` | 管理员为某选手生成随机值 | admin |
| 4 | POST | `/api/performance/admin-generate-all` | 管理员一键为所有选手生成 | admin |
| 5 | GET | `/api/performance/player-status?round=` | 获取各选手是否已生成 | admin |

**请求/响应格式**：

```typescript

//   performanceValue: 12,        // 生成得随机发挥值 (-5 ~ 15)
```
