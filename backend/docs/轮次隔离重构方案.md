# 轮次隔离重构方案

## 一、问题根因分析

### 问题 1：队长数据跨轮次污染

**根因：`User.role` 是全局字段，无轮次概念**

当管理员在第一轮指派队长时 (`POST /api/captain/admin/assign`)，以下代码会修改 `User.role`：

```javascript
// src/routes/captain.js 第145行 — captain.js
if (user && user.role !== 'admin') { user.role = 'captain'; await user.save() }
```

同样，`teams.js` 中通过 `POST /api/teams/:teamId/captain` 设置队长时也会修改 `User.role`。

**后果：** 第一轮队长在第二轮时 `User.role` 仍然是 `'captain'`，前端如果通过 `GET /api/users?role=captain` 查询"已分配队长列表"，会看到所有轮次的队长。

**影响范围：**
- `captain.js` — `/admin/assign` 和 `/admin/unassign` 两个端点
- `teams.js` — `/:teamId/captain` 端点
- `season.js` — `restart` 和 `reset` 端点（重置时已将 role 改回 `player`）

---

### 问题 2：组队页面显示第一轮的分组

**根因：赛季推进时未自动清理旧轮次业务数据**

当从第一轮推进到第二轮时（通过 `POST /api/season/next` 或 `POST /api/season/set`），后端代码**只修改了赛季状态**，没有清除旧轮次的业务数据（队伍、成员、队长、歌曲分配等）。

```javascript
// src/routes/season.js 第148-190行 — season.next
router.post('/next', auth, requireAdmin, async (req, res) => {
    // 只改了 season.currentRound 和 season.currentStage
    // 没有清除旧轮次的队伍/成员/队长等数据
})
```

**后果：** 进入第二轮后，旧轮次的 `RoundTeam` / `RoundTeamMember` / `RoundCaptain` 等数据仍然存在于数据库中。如果前端没传 `roundId`，或后端某些接口在 `roundId` 缺失时返回了默认数据，就会显示旧轮次的数据。

**影响范围：**
- `season.js` — `POST /next` 和 `POST /set` 两个端点

---

### 问题 3：前端可能需要查询"当前轮可用选手列表"

**根因：没有按轮次查询可用选手（未被淘汰、未被分配到队伍的）的接口**

前端在"已选但未分配队长"页面可能使用了 `GET /api/users` 查询所有 `role === 'captain'` 的用户，但 `role` 是全局字段。

---

## 二、改造方案

### 方案 A：后端改造（必须实施）

#### A1. 推进轮次时自动清除旧轮次数据

**修改文件：** `src/routes/season.js`

**目标：** 当管理员推进到新轮次时，自动清除上一轮的业务数据。

**修改 `POST /next` 端点：**

```javascript
router.post('/next', auth, requireAdmin, async (req, res) => {
  try {
    const season = await ensureSeason()

    const nextStage = getNextStage(season.currentStage)
    const prevRound = season.currentRound
    const prevStage = season.currentStage

    if (nextStage === null) {
      // 当前是最后一个阶段（ELIMINATION），自动进入下一轮
      const newRound = season.currentRound + 1
      if (newRound > season.totalRounds) {
        return res.status(400).json({
          success: false,
          error: '已经是最后一轮，无法继续推进',
          code: 'NO_MORE_ROUNDS'
        })
      }

      // ==== 关键：进入新轮次时清除上一轮的业务数据 ====
      const prevRoundDetail = await Round.findOne({ seasonId: season.id, index: prevRound })
      if (prevRoundDetail) {
        const prevRoundId = prevRoundDetail.id
        const prevFrontId = `round-${prevRound}`
        const prevFilter = { roundId: { $in: [prevRoundId, prevFrontId] } }
        
        // 清除旧队伍的队长信息（User.role 不改）
        await RoundTeam.deleteMany(prevFilter)
        await RoundTeamMember.deleteMany(prevFilter)
        await RoundCaptain.deleteMany(prevFilter)
        await CaptainVote.deleteMany(prevFilter)
        await TeamSong.deleteMany(prevFilter)
        await RoundSong.deleteMany({ roundId: prevFrontId })
      }
      // ==== 结束清理 ====

      season.currentRound = newRound
      season.currentStage = 'preparation'
    } else {
      season.currentStage = nextStage
    }
    season.updatedAt = new Date().toISOString()
    await season.save()
    // ... 后续代码保持不变
  }
})
```

**修改 `POST /set` 端点：**

```javascript
router.post('/set', auth, requireAdmin, async (req, res) => {
  try {
    const { round, stage } = req.body
    const season = await ensureSeason()
    const prevRound = season.currentRound
    
    // 如果轮次变化了（进入新轮次），清除旧数据
    if (round !== prevRound) {
      const prevRoundDetail = await Round.findOne({ seasonId: season.id, index: prevRound })
      if (prevRoundDetail) {
        const prevRoundId = prevRoundDetail.id
        const prevFrontId = `round-${prevRound}`
        const prevFilter = { roundId: { $in: [prevRoundId, prevFrontId] } }
        
        await RoundTeam.deleteMany(prevFilter)
        await RoundTeamMember.deleteMany(prevFilter)
        await RoundCaptain.deleteMany(prevFilter)
        await CaptainVote.deleteMany(prevFilter)
        await TeamSong.deleteMany(prevFilter)
        await RoundSong.deleteMany({ roundId: prevFrontId })
      }
    }
    
    season.currentRound = round
    season.currentStage = stage
    season.updatedAt = new Date().toISOString()
    await season.save()
    // ... 后续代码保持不变
  }
})
```

#### A2. 新增按轮次查询可用选手的接口

**修改文件：** `src/routes/captain.js`

**接口：** `GET /api/captain/available-players?roundId=round-1`

**功能：** 返回当前轮中尚未被指派为队长的选手列表（排除已被淘汰的）。

```javascript
router.get('/available-players', auth, async (req, res) => {
  try {
    const { roundId } = req.query
    if (!roundId) return res.status(400).json({ success: false, error: 'roundId 必填', code: 'MISSING_ROUND_ID' })

    const round = await getRound(roundId)
    if (!round) return res.status(400).json({ success: false, error: '轮次不存在', code: 'ROUND_NOT_FOUND' })
    const rId = round.id

    // 获取该轮已被设为队长的选手
    const captains = await RoundCaptain.find({ roundId: rId })
    const captainPlayerIds = new Set(captains.map(c => c.playerId))

    // 获取该轮所有在团队中的选手
    const members = await RoundTeamMember.find({ roundId: rId })
    const memberPlayerIds = new Set(members.map(m => m.playerId))

    // 获取所有活跃选手
    const users = await User.find({})
    const available = users.filter(u => {
      if (u.role === 'admin') return false              // 排除管理员
      if (u.status === 'eliminated') return false        // 排除已淘汰
      return !captainPlayerIds.has(u.id)                 // 排除已在本轮被指派的队长
    })

    res.json({
      success: true,
      data: available.map(u => ({
        id: u.id,
        name: u.name,
        avatar: u.avatar || null,
        attributes: u.attributes,
        inTeam: memberPlayerIds.has(u.id),               // 是否已在该轮的队伍中
        status: u.status
      }))
    })
  } catch (e) {
    console.error(e)
    res.status(500).json({ success: false, error: '获取可用选手失败', code: 'SERVER_ERROR' })
  }
})
```

#### A3. 新增查询当前轮已分配队长列表的接口

**接口：** `GET /api/captain/current?roundId=round-1`

**功能：** 返回本轮所有已分配的队长（队长+所属队伍）。

```javascript
router.get('/current', auth, async (req, res) => {
  try {
    const { roundId } = req.query
    if (!roundId) return res.status(400).json({ success: false, error: 'roundId 必填', code: 'MISSING_ROUND_ID' })
    const round = await getRound(roundId)
    if (!round) return res.status(400).json({ success: false, error: '轮次不存在', code: 'ROUND_NOT_FOUND' })
    const rId = round.id

    // 只查该轮的队长
    const captains = await RoundCaptain.find({ roundId: rId })
    const teams = await RoundTeam.find({ roundId: rId })
    const users = await User.find({})
    const teamMap = {}
    const userMap = {}
    for (const t of teams) teamMap[t.id] = t
    for (const u of users) userMap[u.id] = u

    const enriched = captains.map(c => ({
      id: c.id,
      playerId: c.playerId,
      playerName: userMap[c.playerId]?.name || null,
      playerAvatar: userMap[c.playerId]?.avatar || null,
      teamId: c.teamId,
      teamName: teamMap[c.teamId]?.name || null,
    }))

    res.json({ success: true, data: enriched, total: enriched.length })
  } catch (e) {
    console.error(e)
    res.status(500).json({ success: false, error: '获取队长列表失败', code: 'SERVER_ERROR' })
  }
})
```

### 方案 B：前端修复（配合实施）

#### B1. 队长查询改用新接口

**当前做法（有问题的）：**
```typescript
// 问题：查询所有 role === 'captain' 的用户，会跨轮次污染
const { data } = await api.get('/api/users', { params: { role: 'captain' } })
```

**修复后：**
```typescript
// 正确：按轮次查询已分配的队长
const { data } = await api.get('/api/captain/current', { params: { roundId: `round-${currentRound}` } })

// 正确：查询当前轮可用的队长候选人
const { data } = await api.get('/api/captain/available-players', { params: { roundId: `round-${currentRound}` } })
```

前端在所有涉及轮次数据的 API 调用中，都必须传递 `roundId` 参数。以下是所有需要传 `roundId` 的接口清单：

### 方案 C：后端自动清理（按轮次推进触发）

当管理员推进到新轮次时，以下数据会自动清理：

| 数据表 | 清理范围 | 说明 |
|---|---|---|
| `RoundTeam` | 上一轮所有队伍 | 新轮次需要重新分配阵容 |
| `RoundTeamMember` | 上一轮所有成员关系 | 选手在新轮次重新组队 |
| `RoundCaptain` | 上一轮的所有队长记录 | 新轮次重新选举/指派 |
| `CaptainVote` | 上一轮的所有投票 | 新轮次重新投票 |
| `TeamSong` | 上一轮的所有队伍选歌 | 新轮次重新选歌 |
| `RoundSong` | 上一轮的所有歌曲清单 | 新轮次重新配置歌曲 |
| `RoundPreparation` | 上一轮的预先准备配置 | 新轮次重新配置 |
| `PerformanceValue` | 上一轮的发挥值 | 新轮次重新生成 |
| `PerformanceRoundState` | 上一轮的公演状态 | 新轮次重新开启 |

**注意事项：**
- `User.role` **不清理** — 因为队长角色会被新轮次的指派覆盖
- `TeamPerformance` / `PlayerPerformance` / `Elimination` **不清理** — 公演结果和淘汰记录需要保留作为历史查看
- `TrainingRecord` / `RehearsalResult` **不清理** — 训练和彩排记录保留

## 四、实施顺序

1. **先实施后端（方案 A）**：不改后端之前，前端改也没用
   - A1：`/next` 和 `/set` 推进时自动清理
   - A2：`/available-players` 按轮次查可用选手
   - A3：`/current` 按轮次查已分配队长

2. **再实施前端（方案 B）**：后端改完之后，前端调整调用方式
   - B1：改用新接口查询队长
   - B2：所有接口补上 `roundId`

3. **重启测试**：确保所有功能正常
