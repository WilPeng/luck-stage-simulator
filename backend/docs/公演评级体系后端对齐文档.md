# 公演评级体系 — 后端对齐文档

## 一、评级计算公式

### 选手个人得分

```
normalizedVocalW   = song.vocalWeight  / (vocalWeight + danceWeight + charmWeight)
normalizedDanceW   = song.danceWeight  / total
normalizedCharmW   = song.charmWeight  / total

attributeScore     = player.vocal × normalizedVocalW
                   + player.dance × normalizedDanceW
                   + player.charm × normalizedCharmW
                   // 范围: 0~100

difficultyFactor   = 1 - (song.difficulty - 1) × 0.1
                   // difficulty=1 → 1.0, difficulty=5 → 0.6

performanceBonus   = player生成的随机发挥值 × 2
                   // 发挥值范围 -5~15, 加分范围 -10~30

playerFinalScore   = Math.round(attributeScore × difficultyFactor + performanceBonus)
                   // 最终范围 0~120（可截断）
```

### 选手评级映射

| 评级 | 名称 | 分数阈值 |
|------|------|---------|
| S | 完美舞台 | ≥ 85 |
| A | 出色表现 | ≥ 65 |
| B | 稳定发挥 | ≥ 45 |
| C | 略有不足 | ≥ 25 |
| D | 失误较多 | < 25 |

### 团队得分

```
teamAvgScore  = round(average of all member's playerFinalScore)
teamBonus     = random(0, 20)  (团队配合随机加成)
teamScore     = teamAvgScore + teamBonus
```

团队评级使用与选手相同的阈值映射。

### 最终票数（用于排名）

```
finalVotes = 500 + teamScore × 3 + random(-10, 20)
```

## 二、新增接口

### 阶段一：选手发挥（5个新接口）

#### 1. 管理员开启公演

```
POST /api/performance/start
权限: admin
Body: { "round": 1 }
Response: { "success": true, "message": "公演已开启" }
```

后端逻辑：标记该轮次为"公演进行中"状态，选手端获得操作权限。

#### 2. 选手生成发挥值

```
POST /api/performance/player-generate
权限: player（需登录）
Body: { "round": 1 }
Response: {
  "success": true,
  "playerId": "u001",
  "performanceValue": 12        // 随机值 -5 ~ 15
}
```

后端逻辑：
- 检查该轮次是否已开启
- 为选手生成一个 -5 到 15 之间的随机整数
- 保存该值到数据库
- 每个选手每轮只能生成一次

#### 3. 管理员代生成

```
POST /api/performance/admin-generate
权限: admin
Body: { "round": 1, "playerId": "u001" }
Response: 同上
```

#### 4. 管理员一键全生成

```
POST /api/performance/admin-generate-all
权限: admin
Body: { "round": 1 }
Response: { "success": true, "generatedCount": 33 }
```

后端逻辑：为该轮所有尚未生成的选手生成随机值。

#### 5. 获取选手发挥状态

```
GET /api/performance/player-status?round=1
权限: admin
Response: {
  "totalPlayers": 33,
  "generatedCount": 20,
  "players": [
    { "playerId": "u001", "playerName": "张三", "teamId": "round-1-team-1", "teamName": "星光队", "generated": true, "performanceValue": 12 },
    { "playerId": "u002", "playerName": "李四", "teamId": "round-1-team-1", "teamName": "星光队", "generated": false, "performanceValue": null }
  ]
}
```

### 阶段二：公演结算（已有）

```
POST /api/performance/calculate
Body: { "roundId": "round-1" }
```

使用上面的评级公式计算，返回结果中需要新增字段：

**PlayerPerformanceResult 新增字段**：
```json
{
  "playerScore": 78,          // 选手个人得分 0~120
  "stageRating": "A",         // S/A/B/C/D
  "stageRatingText": "出色表现"
}
```

**TeamPerformanceResult 新增字段**：
```json
{
  "teamScore": 82,            // 团队综合得分
  "teamRating": "A",          // S/A/B/C/D
  "teamRatingText": "出色表现"
}
```

## 三、字段变更汇总

### PlayerPerformanceResult

| 字段 | 类型 | 说明 |
|------|------|------|
| `playerScore` | number | 选手个人得分 0~120 |
| `stageRating` | string | S/A/B/C/D |
| `stageRatingText` | string | 评级描述（完美舞台/出色表现/稳定发挥/略有不足/失误较多） |

### TeamPerformanceResult

| 字段 | 类型 | 说明 |
|------|------|------|
| `teamScore` | number | 团队综合得分 |
| `teamRating` | string | S/A/B/C/D |
| `teamRatingText` | string | 评级描述 |

### GenerateAudienceVoteRequest

| 字段 | 类型 | 说明 |
|------|------|------|
| `round` | number | 轮次序号（新加，兼容旧字段roundId） |
| `roundId` | string | 轮次ID（保留兼容） |
| `weights` | array | 选手权重数组（可选，管理员编辑后的权重） |

## 四、完整数据流

```
管理员点击"选手开始公演"
  → POST /api/performance/start { round: 1 }

选手端点击生成
  → POST /api/performance/player-generate { round: 1 }
  → 后端返回随机值 performanceValue (-5~15)

管理员可代生成
  → POST /api/performance/admin-generate / admin-generate-all

全员就绪后，管理员点击"开始结算"
  → POST /api/performance/calculate { roundId: "round-1" }
  → 后端使用评级公式计算：
      选手个人分 = attributeScore × difficultyFactor + performanceBonus × 2
      选手评级  = f(选手个人分) → S/A/B/C/D
      团队分    = avg(成员个人分) + random(0,20)
      团队评级  = f(团队分) → S/A/B/C/D
      最终票数  = 500 + teamScore × 3 + random(-10,20)
  → 返回 teamResults（含评级）+ playerResults（含评级）

管理员逐一揭晓队伍
  → 将队伍 status 从 'calculated' 改为 'confirmed'
  → 显示成员个人评级 + 团队评级 + 最终票数

全部揭晓后，进入喜爱度票数标签
  → 编辑权重 → POST /api/admin/audience-vote/generate
  → 微调评审投票 → PUT /api/admin/audience-vote/seat/:num
  → 释放结果 → POST /api/admin/audience-vote/release
  → 进入淘汰
```
