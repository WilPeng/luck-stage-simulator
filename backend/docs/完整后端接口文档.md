# 乘风2026运气赛 - 完整后端接口文档

**服务地址**: `http://localhost:3000`
**API前缀**: `/api`
**认证方式**: Bearer Token（登录接口返回 `token`，后续请求需在 Header 中携带 `Authorization: Bearer <token>`）
**响应格式**: `{ success: boolean, data: any, message?: string, code?: string }`
**错误码**: `400` 参数错误 / `401` 未认证 / `403` 无权限 / `404` 资源不存在 / `500` 服务器错误

---

## 3. 队伍管理模块 `/api/teams`

> **核心原则**: 所有操作均按 `round` 轮次隔离。前端各模块顶部选择当前轮次，所有请求传 `round` 参数。

### 3.1 获取队伍列表
- **方法**: `GET` `/api/teams?round=1`
- **说明**: 不传 `round` 返回所有轮次，建议始终传

### 3.2 队伍统计
- **方法**: `GET` `/api/teams/stats/summary?round=1`

### 3.3 配置队伍结构（初始化/重建）⭐
- **方法**: `POST` `/api/teams/setup`
- **权限**: 管理员
- **请求体**:
  ```json
  {
    "round": 1,
    "teamCount": 3,
    "teamSizes": [5, 4, 6],
    "teamNames": ["乘风一团", "乘风二团", "乘风三团"]
  }
  ```
- **说明**: 各队人数可以不同。调用后会清空该轮原有队伍结构。

### 3.4 新增单支队伍
- **方法**: `POST` `/api/teams`
- **请求体**: `{ "name", "maxMembers", "round" }`

### 3.5 自动分配选手 ⭐ 修改
- **方法**: `POST` `/api/teams/auto-distribute`
- **请求体**: `{ "round": 1 }`
- **逻辑**: 保留各队已有成员和队长；剩余未分配选手随机打乱后逐个随机入有空位的队

### 3.6 添加成员
- **方法**: `POST` `/api/teams/:teamId/members/add` body: `{ "userId" }`

### 3.7 移除成员
- **方法**: `POST` `/api/teams/:teamId/members/remove` body: `{ "userId" }`

### 3.8 交换成员
- **方法**: `POST` `/api/teams/:teamId/members/swap` body: `{ "userId", "otherTeamId", "otherUserId" }`

### 3.9 设置队长
- **方法**: `POST` `/api/teams/:teamId/captain` body: `{ "userId" }`

### 3.10 锁定/解锁
- **方法**: `POST` `/api/teams/:teamId/lock` / `/unlock`

## 5. 歌曲管理模块 `/api/songs`

> **核心变更**: 歌曲类型有四种，歌曲分配给**选手**而不是队伍，按轮次隔离。
> **重要**: 歌曲管理分为两个阶段：
> 1. **选择本轮歌曲**（RoundSong）：管理员选择本轮公演使用的歌曲及其给分方式
> 2. **歌曲分配**（SongAssignment）：将歌曲分配给具体选手/队伍

### 歌曲类型说明

| type | 名称 | 约束 |
|------|------|------|
| `team_show` | 团秀 | 同一团全部成员，建议每团一个 |
| `team_collab` | 团内合作秀 | 同团部分成员 |
| `captain_show` | 队长秀 | 仅当轮队长 |
| `pk_show` | PK秀 | 不限团队 |

### 5.1 获取歌曲列表
- **方法**: `GET` `/api/songs?type=team_show`
- **说明**: 支持按 `type` 筛选

### 5.2 新增歌曲
- **方法**: `POST` `/api/songs`
- **请求体**:
  ```json
  {
    "name": "逆光飞翔",
    "style": "流行",
    "difficulty": 3,
    "vocalWeight": 4,
    "danceWeight": 3,
    "charmWeight": 3,
    "baseScore": 100,
    "riskFactor": 0.2,
    "type": "team_show"
  }
  ```

### 5.3 批量新增
- **方法**: `POST` `/api/songs/batch`

### 5.4 更新歌曲
- **方法**: `PUT` `/api/songs/:id`

### 5.5 删除歌曲
- **方法**: `DELETE` `/api/songs/:id`

### 5.6 单条分配歌曲 ⭐（分配给选手）
- **方法**: `POST` `/api/songs/assign`
- **请求体**:
  ```json
  {
    "round": 1,
    "songId": "歌曲ID",
    "userIds": ["选手1ID", "选手2ID", "选手3ID"],
    "teamId": "团队ID",
    "songType": "team_show"
  }
  ```
- **约束校验**:
  - `team_show` / `team_collab`: 所有 `userIds` 必须同属 `teamId` 对应的团
  - `songType` 优先使用前端传入值，其次用歌曲自带的 `type`

### 5.7 批量分配歌曲 ⭐
- **方法**: `POST` `/api/songs/batch-assign`
- **请求体**:
  ```json
  {
    "round": 1,
    "assignments": [
      { "songId": "歌1", "teamId": "团1", "userIds": ["u1","u2","u3"], "songType": "team_show" },
      { "songId": "歌2", "teamId": "团1", "userIds": ["u1","u2"], "songType": "team_collab" },
      { "songId": "歌3", "teamId": "团2", "userIds": ["captainId"], "songType": "captain_show" },
      { "songId": "歌4", "teamId": null, "userIds": ["u1","u10"], "songType": "pk_show" }
    ]
  }
  ```
- **空数组语义**: `assignments` 为空数组 `[]` 时，表示**清空该轮所有歌曲分配**，同时删除对应的 RoundSong 记录
- **说明**: 调用后先清空该轮原有分配，再写入新的

### 5.8 查询某轮歌曲分配列表 ⭐
- **方法**: `GET` `/api/songs/assignments/list?round=1`
- **说明**: 返回分配详情（含歌曲信息、类型、选手列表）
- **响应**:
  ```json
  {
    "success": true,
    "data": [
      {
        "id": "分配ID",
        "round": 1,
        "songId": "歌曲ID",
        "songType": "team_show",
        "teamId": "团队ID",
        "userIds": ["u1","u2","u3"],
        "song": { "id", "name", "type", "baseScore", ... },
        "users": [
          { "userId": "u1", "name": "小王" },
          { "userId": "u2", "name": "小李" }
        ]
      }
    ]
  }
  ```
- **前端展示建议**: 按团队分组，每团下展示其所有歌曲，团秀单独高亮

### 5.9 删除单个分配
- **方法**: `DELETE` `/api/songs/assignments/:id`

### 5.10 验证本轮歌曲选择 ⭐ 新增
- **方法**: `POST` `/api/songs/validate-round-selection`
- **请求体**:
  ```json
  {
    "round": 1,
    "teamShowSongs": ["song1", "song2", "song3"],
    "teamCollabSongs": [],
    "captainShowSong": null,
    "pkShowSongs": ["song7"]
  }
  ```
- **验证规则**:
  - 团秀：必须选择与队伍数量相同的歌曲
  - 合作秀：0 首或与队伍数量相同的歌曲
  - 队长秀：0 首或 1 首
  - PK秀：0 首到队伍数量之间的歌曲
- **响应**:
  ```json
  { "success": true, "data": { "valid": true, "errors": [] } }
  ```

### 5.11 轮次歌曲管理 `/api/songs/round` ⭐⭐ 新增

> **说明**: 此模块管理"本轮选用哪些歌曲"，与 SongAssignment（歌曲分配给选手）是两个独立阶段。
> **RoundSong** 记录本轮公演选用的歌曲及其给分方式；**SongAssignment** 记录每首歌分配给哪些人/队。
> 清空 RoundSong 时会同步清空对应的 SongAssignment。

#### 5.11.1 批量添加歌曲到本轮
- **方法**: `POST` `/api/songs/round/batch`
- **权限**: 管理员
- **请求体**:
  ```json
  {
    "round": 1,
    "songs": [
      {
        "songId": "歌曲ID",
        "songType": "team_show",
        "scoringMethod": "actual"
      },
      {
        "songId": "歌曲ID",
        "songType": "captain_show",
        "scoringMethod": "ranked"
      }
    ]
  }
  ```
- **字段说明**:
  - `songType`: `team_show` | `team_collab` | `captain_show` | `pk_show`
  - `scoringMethod`: `actual`（按实际得分）| `fixed`（固定分数）| `ranked`（按排名给分）
  - `scoringMethod` 对团秀/合作秀可选（默认 actual），对队长秀/PK秀必填
- **响应**: `{ "success": true }`

#### 5.11.2 获取本轮所有歌曲
- **方法**: `GET` `/api/songs/round/:round`
- **说明**: 返回该轮所有已选歌曲（含歌曲详情）
- **响应**:
  ```json
  {
    "success": true,
    "data": [
      {
        "id": "roundSongId",
        "round": 1,
        "songId": "歌曲ID",
        "songType": "team_show",
        "scoringMethod": "actual",
        "song": {
          "id": "歌曲ID",
          "name": "逆光飞翔",
          "type": "team_show",
          "difficulty": 3,
          "baseScore": 100,
          ...
        },
        "createdAt": "ISO时间"
      }
    ]
  }
  ```

#### 5.11.3 获取本轮歌曲（含分配信息）
- **方法**: `GET` `/api/songs/round/:round/full`
- **说明**: 返回歌曲 + 分配信息，用于展示本轮公演每首歌的参赛人员
- **响应**:
  ```json
  {
    "success": true,
    "data": [
      {
        "id": "roundSongId",
        "round": 1,
        "songId": "歌曲ID",
        "songType": "team_show",
        "scoringMethod": "actual",
        "song": { ... },
        "assignment": {
          "id": "assignmentId",
          "teamId": "团队ID",
          "teamName": "乘风一团",
          "userIds": ["u1", "u2"],
          "users": [
            { "userId": "u1", "name": "小王" },
            { "userId": "u2", "name": "小李" }
          ]
        },
        "createdAt": "ISO时间"
      }
    ]
  }
  ```

#### 5.11.4 移除轮次歌曲
- **方法**: `DELETE` `/api/songs/round/:id`
- **权限**: 管理员
- **说明**: 删除时会同步清空对应的 SongAssignment
- **响应**: `{ "success": true }`

#### 5.11.5 更新给分方式
- **方法**: `PUT` `/api/songs/round/:id/scoring`
- **权限**: 管理员
- **请求体**:
  ```json
  { "scoringMethod": "ranked" }
  ```
- **响应**: 返回更新后的 RoundSong 对象

#### 5.11.6 清空本轮所有歌曲
- **方法**: `DELETE` `/api/songs/round/clear/:round`
- **权限**: 管理员
- **说明**: 清空该轮所有 RoundSong 及对应的 SongAssignment
- **响应**: `{ "success": true }`

## 6. 训练管理模块 `/api/training`

> **核心原则**: 训练次数按 `round` 隔离，从 TrainingRecord 统计而非 user.trainingCount。

### 6.1 获取训练卡牌列表
- **方法**: `GET` `/api/training/cards`

### 6.2 新增卡牌
- **方法**: `POST` `/api/training/cards`

### 6.3 更新卡牌
- **方法**: `PUT` `/api/training/cards/:id`

### 6.4 删除卡牌
- **方法**: `DELETE` `/api/training/cards/:id`

### 6.5 重置为预设卡牌
- **方法**: `POST` `/api/training/cards/reset-presets`

### 6.6 获取训练配置 ⭐
- **方法**: `GET` `/api/training/config`
- **响应**: `{ "drawsPerPlayer": 5, "currentRound": 1 }`
- **注意**: 已移除 `totalRounds`

### 6.7 更新训练配置 ⭐
- **方法**: `PUT` `/api/training/config`
- **请求体**: `{ "drawsPerPlayer"?, "currentRound"? }`

### 6.8 训练统计 ⭐
- **方法**: `GET` `/api/training/stats?round=1`
- **响应**:
  ```json
  {
    "success": true,
    "data": {
      "round": 1,
      "totalPlayers": 15,
      "completedPlayers": 10,
      "uncompletedPlayers": 5,
      "totalDraws": 35,
      "maxDrawsPerPlayer": 5,
      "currentRound": 1,
      "completionRate": 67,
      "uncompletedList": [
        { "id", "name", "trainingCount": 0, "completed": false }
      ]
    }
  }
  ```
- **说明**: `trainingCount` 由该轮 TrainingRecord 统计而来

### 6.9 单人抽卡 ⭐
- **方法**: `POST` `/api/training/draw/:userId`
- **Body**（可选）: `{ "round": 1 }`
- **逻辑**: 按 `round` 判断次数上限，记录也带 `round`

### 6.10 一键完成训练 ⭐⭐ 修改
- **方法**: `POST` `/api/training/auto-complete-all`
- **Body**（可选）: `{ "round": 1 }`
- **新逻辑**:
  - 只处理**本轮训练次数 == 0** 的选手（未开始训练的）
  - 已训练过但未满次数的选手**完全跳过**，不会补剩余
  - 被处理的选手**随机抽 1 到 drawsPerPlayer 次**
- **响应**:
  ```json
  {
    "success": true,
    "data": {
      "round": 1,
      "processedCount": 3,
      "totalDraws": 11,
      "results": [
        { "playerId", "playerName", "drawsCount": 4, "draws": [...], "finalAttributes": {...} }
      ]
    }
  }
  ```

### 6.11 为指定选手补齐训练 ⭐
- **方法**: `POST` `/api/training/auto-complete/:userId`
- **Body**（可选）: `{ "round": 1 }`
- **说明**: 按 `round` 判断该选手是否未开始或未满次数

### 6.12 获取训练记录 ⭐
- **方法**: `GET` `/api/training/records?round=1&page=1&pageSize=1000`
- **响应**:
  ```json
  {
    "success": true,
    "data": {
      "list": [
        { "id", "userId", "userName", "cardName", "cardType", "effect", "attributesAfter", "round": 1, "createdAt" }
      ],
      "total": 156,
      "page": 1,
      "pageSize": 1000,
      "totalPages": 1
    }
  }
  ```

### 6.13 获取指定选手训练记录
- **方法**: `GET` `/api/training/records/user/:userId`

  ```json
  { "round": 1, "teamIds": ["teamId1", "teamId2"] }
  ```
- **说明**: 覆盖式保存，每次调用先清空该轮旧标记，再写入新的
- **响应**:
  ```json
  {
    "success": true,
    "data": [
      { "id": "xxx", "round": 1, "teamId": "teamId1", "teamName": "乘风一团", "markedAt": "..." }
  ```

## 11. 聊天模块 `/api/chat`

> **设计说明**: 简单非实时聊天，所有已登录用户（管理员+选手）均可发消息、查看历史。消息按时间倒序展示，无在线状态/实时推送，前端可自行轮询刷新。

### 11.1 获取聊天记录
- **方法**: `GET` `/api/chat/messages`
- **权限**: 已登录用户（管理员/选手均可）
- **查询参数**:
  - `page`: 页码，默认 `1`
  - `pageSize`: 每页条数，默认 `50`
  - `keyword`: 关键词搜索（匹配消息内容和发送者昵称），可选
- **响应**:
  ```json
  {
    "success": true,
    "data": {
      "list": [
        {
          "id": "消息ID",
          "senderId": "发送者ID",
          "senderName": "小王",
          "senderRole": "player",
          "senderAvatar": "/uploads/avatars/avatar-xxx.png",
          "content": "大家加油！",
          "createdAt": "2026-06-13T14:30:00.000Z"
        }
      ],
      "total": 156,
      "page": 1,
      "pageSize": 50,
      "totalPages": 4
    }
  }
  ```

### 11.2 发送消息
- **方法**: `POST` `/api/chat/messages`
- **权限**: 已登录用户
- **请求体**:
  ```json
  { "content": "大家加油！" }
  ```
- **响应**: 返回创建的消息对象（含 id、createdAt）
  ```json
  {
    "success": true,
    "data": {
      "id": "消息ID",
      "senderId": "发送者ID",
      "senderName": "小王",
      "senderRole": "player",
      "content": "大家加油！",
      "createdAt": "2026-06-13T14:30:00.000Z"
    }
  }
  ```

### 11.3 删除消息
- **方法**: `DELETE` `/api/chat/messages/:id`
- **权限**: 消息发送者本人 或 管理员
- **响应**: `{ "success": true, "data": { "id": "消息ID" } }`

### 11.4 清空所有聊天记录（谨慎操作）
- **方法**: `DELETE` `/api/chat/messages`
- **权限**: 仅管理员

### 11.5 获取未读消息数
- **方法**: `GET` `/api/chat/messages/unread-count`
- **查询参数**: `since`: ISO 时间字符串（上次拉取时间）
- **响应**: `{ "success": true, "data": { "count": 5 } }`
- **说明**: 返回 `since` 时间之后的新消息数量，前端用于红点提示
