## 1. 赛季管理 (Season)

文件：`/api/season`

| 方法 | 路径 | 权限 | 功能 | 说明 |
|------|------|------|------|------|
| GET | `/api/season` | auth | 获取当前赛季信息 | |
| GET | `/api/season/rounds` | auth | 获取该赛季的轮次列表 | |
| GET | `/api/season/stages` | auth | 获取阶段列表 | |
| POST | `/api/season/set` | admin | 管理员手动设置进度（轮次+阶段） | 轮次变化时自动清理上一轮业务数据 |
| POST | `/api/season/next` | admin | 自动推进到下一阶段 | ELIMINATION→下一轮时自动清理 |
| GET | `/api/season/progress/matrix` | auth | 获取赛程矩阵 | |
| GET | `/api/season/progress` | auth | 获取赛程进度（含矩阵+阶段映射） | |
| GET | `/api/season/menu` | auth | 获取菜单权限列表 | |
| PUT | `/api/season/round` | admin | 设置总轮次和每轮训练次数 | |
| PUT | `/api/season/config` | admin | 更新赛季配置 | |
| POST | `/api/season/restart` | admin | 一键重新开始（保留历史公演和淘汰记录） | |
| POST | `/api/season/reset` | admin | 完全重置（清空所有业务数据） | |
| GET | `/api/season/history` | auth | 获取指定轮次的历史数据 | 参数：`roundId`（必填） |

**POST /api/season/set 请求体：**
```json
{ "round": 1, "stage": "preparation" }
```

**POST /api/season/restart 清理的数据：**
RoundTeam、RoundTeamMember、RoundCaptain、CaptainVote、TeamSong、RoundSong、RoundPreparation、PerformanceRoundState、PerformanceValue、Team、TrainingRecord、RehearsalResult

**不清除的数据：**
User（用户信息）、Song（歌曲库）、TrainingCard（训练卡牌库）

文件：`/api/songs`

> 独立于轮次的全局歌曲库 CRUD 管理，数据不会因轮次推进被清理。

| 方法 | 路径 | 权限 | 功能 | 说明 |

| POST | `/api/songs` | admin | 新增歌曲 | |
| PUT | `/api/songs/:id` | admin | 更新歌曲 | |
| DELETE | `/api/songs/:id` | admin | 删除歌曲 | |

### 歌曲模型字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| id | string | 自动生成 | 唯一标识 |
| name | string | - | 歌名 |
| difficulty | number | 3 | 难度（1~5） |
| vocalWeight | number | 3 | 声乐权重 |
| danceWeight | number | 3 | 舞蹈权重 |
| charmWeight | number | 3 | 魅力权重 |
| type | string | group | solo/duet/group |
| style | string | 流行 | 风格 |

| riskFactor | number | 0.2 | 风险系数 |

## 8. 训练卡牌管理 (Training Card)

文件：`/api/training`

> 独立于轮次的全局卡牌库 CRUD 管理，数据不会因轮次推进被清理。

| 方法 | 路径 | 权限 | 功能 |
|------|------|------|------|
| GET | `/api/training/cards` | auth | 获取训练卡列表 |
| POST | `/api/training/cards` | admin | 新增训练卡 |
| PUT | `/api/training/cards/:id` | admin | 更新训练卡 |
| DELETE | `/api/training/cards/:id` | admin | 删除训练卡 |

### 卡牌模型字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| id | string | 自动生成 | 唯一标识 |
| name | string | - | 卡牌名称 |
| type | string | mixed | 类型（vocal/dance/charm/mixed 等） |
| description | string | '' | 描述 |
| effect | object | { vocal: 1 } | 属性效果 |
| weight | number | 10 | 权重（加权随机抽卡用） |
| enabled | boolean | true | 是否启用 |

**effect 支持的字段：**
- `vocal` - 增加声乐属性
- `dance` - 增加舞蹈属性
- `charm` - 增加魅力属性
- `randomOne` - 随机增加一个属性
- `randomTwo` - 随机增加两个属性
- `lowest` - 增加最低的那个属性
- `highest` - 增加最高的那个属性

**POST /api/training/cards 请求体：**
```json
{
  "name": "声乐特训",
  "type": "vocal",
  "description": "提升声乐属性",
  "effect": { "vocal": 3 },
  "weight": 10,
  "enabled": true
}
```

**GET /api/training/cards 返回格式：**
```json
{
  "success": true,
  "data": [
    {
      "id": "card_xxx",
      "name": "声乐特训",
      "type": "vocal",
      "description": "提升声乐属性",
      "effect": { "vocal": 3 },
      "weight": 10,
      "enabled": true,
      "createdAt": "2026-06-24T...",
      "updatedAt": "2026-06-24T..."
    }
  ],
  "total": 15
}
```

说明：
- 管理员可以看到所有卡牌（包括 enabled: false 的）
- 非管理员只能看到 enabled !== false 的卡牌
- 按名称字母排序

## 12. 公演结算与得分规则

### 个人分计算

```
个人最终分 = 属性分 × 难度系数 + 发挥加成（截取到 0~120）
```

**① 属性分（0~100）**
```
属性分 = vocal × vocal占比 + dance × dance占比 + charm × charm占比
各项占比 = 该项权重 / (vocalWeight + danceWeight + charmWeight)
```

**② 难度系数**（范围 1.0 ~ 0.6）
```
难度系数 = 1 - (难度 - 1) × 0.1
难度1→1.0, 难度2→0.9, 难度3→0.8, 难度4→0.7, 难度5→0.6
```

**③ 发挥加成**（范围 -10 ~ 30）
```
发挥加成 = 发挥值 × 2
发挥值范围 -5 ~ 15
```

**④ 评级判定**

| 分数 | 评级 | 文本 |
|------|------|------|
| ≥ 85 | S | 完美舞台 |
| ≥ 65 | A | 出色表现 |
| ≥ 45 | B | 稳定发挥 |
| ≥ 25 | C | 略有不足 |
| < 25 | D | 失误较多 |

### 团队分计算

```
团队分 = 队员个人分平均值 + 随机团队加成(0~20)
```

### 最终票数（排名依据）

```
最终票数 = 500(基础票) + 团队分×3 + random(-10, 20)
```

| 字段 | 计算方式 |
|------|----------|
| baseVotes | 固定 500 |
| attributeVotes | 团队分 × 3 |
| performanceVotes | random(-10, 20) |
| eventVotes | 团队加成（0~20 随机，即 teamBonus） |
| compatibilityVotes | 始终为 0（预留） |

### 排名规则

- **队伍排名**：按 `finalVotes` 降序
- **选手个人排名（全场）**：按 `playerScore` 降序
- **选手队内排名**：按 `playerScore` 在队内降序
- **选手淘汰排名（按个人喜爱度）**：见第14节

## 17. 认证 (Auth)

文件：`/api/auth`

| 方法 | 路径 | 权限 | 功能 |
|------|------|------|------|
| POST | `/api/auth/login` | 公开 | 用户通过登录码登录，返回 JWT token | 登录成功后服务端会标记该用户 `hasLogin = true` |
| GET | `/api/auth/me` | token | 获取当前登录用户信息 | 返回包含 `hasLogin` 字段的 User 对象 |
| POST | `/api/auth/logout` | token | 用户登出 | 必须调用此接口，服务端会清除 `hasLogin` 状态 |

### 17.1 登录

**POST /api/auth/login**

请求体：
```json
{
  "code": "ABC123"
}
```

响应：
```json
{
  "success": true,
  "data": {
    "user": {
      "id": "player-1",
      "name": "张三",
      "role": "player",
      "status": "active",
      "hasLogin": true,
      "loginCode": "ABC123",
      "attributes": { "vocal": 60, "dance": 50, "charm": 45 },
      "trainingCount": 0
    },
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

服务端行为：
- 设置该用户的 `hasLogin = true`
- 返回 JWT token，后续请求需在 Header 中携带 `Authorization: Bearer {token}`

### 17.2 退出登录

**POST /api/auth/logout**

请求头：`Authorization: Bearer {token}`

响应：
```json
{
  "success": true
}
```

服务端行为：
- 设置当前用户的 `hasLogin = false`
- 使当前 token 失效

### 17.3 切换账号流程（重要）

**前端必须遵循的调用顺序：**

```
正确：
  1. POST /api/auth/logout        → 清除旧用户的 hasLogin 状态
  2. POST /api/auth/login         → 登录新用户，获取新 token

错误：
  1. POST /api/auth/login          → 直接登录新用户
  2. (旧会话后台超时后 logout)      → 错误地清除了新用户的 hasLogin
```

**关键说明：**

- **禁止跳过 logout**：切换账号时必须先调 `logout` 再调 `login`，不能直接调 `login` 覆盖
- **为什么要先 logout**：服务端用 `hasLogin` 字段追踪用户登录状态。如果跳过 logout 直接 login，旧会话的 logout 请求在网络超时后仍可能执行，导致将新用户的 `hasLogin` 错误地置为 `false`
- **token 管理**：logout 成功后客户端应清除本地保存的旧 token；login 成功后保存新 token
- **重试机制**：如果 logout 请求因网络原因失败（非超时），应提示用户重试，不可直接跳入 login

**前端切换账号的推荐伪代码：**

```typescript
async function switchAccount(newCode: string) {
  // 第一步：先登出，清除服务端状态
  try {
    await api.post('/auth/logout')
  } catch (e) {
    // 网络出错时提示用户
    showError('登出失败，请稍后重试')
    return
  }
  
  // 第二步：清除本地 token
  localStorage.removeItem('token')
  localStorage.removeItem('currentUser')
  
  // 第三步：登录新用户
  const { token, user } = await api.post('/auth/login', { code: newCode })
  localStorage.setItem('token', token)
  localStorage.setItem('currentUser', JSON.stringify(user))
  
  // 第四步：跳转到首页
  router.replace(user.role === 'admin' ? '/admin/dashboard' : '/player/home')
}
```

## 20. 轮次隔离说明

### 数据自动清理时机

进入新轮次时（通过 `POST /api/season/next` 或 `POST /api/season/set`），以下数据会被自动清除：

| 数据表 | 说明 |
|--------|------|
| RoundTeam | 上一轮所有队伍 → 新轮次重新分配阵容 |
| RoundTeamMember | 上一轮所有成员关系 → 选手在新轮次重新组队 |
| RoundCaptain | 上一轮所有队长记录 → 新轮次重新选举/指派 |
| CaptainVote | 上一轮所有投票 → 新轮次重新投票 |
| TeamSong | 上一轮所有队伍选歌 → 新轮次重新选歌 |
| RoundSong | 上一轮所有歌曲清单 → 新轮次重新配置歌曲 |

### 不会被清理的数据

| 数据 | 说明 |
|------|------|
| User | 选手/管理员信息（永久保留） |
| Song | 歌曲库（永久保留） |
| TrainingCard | 训练卡牌库（永久保留） |
| TeamPerformance / PlayerPerformance | 公演结果（保留历史可查看） |
| Elimination | 淘汰记录（保留历史可查看） |
| TrainingRecord / RehearsalResult | 训练和彩排记录（保留历史） |

### 前端需要配合的改动

1. **所有涉及轮次的接口调用必须传 roundId**，确保查询按轮次隔离
2. **队长查询**：改用 `GET /api/captain/current?roundId=round-X` 替代 `GET /api/users?role=captain`
3. **可用选手查询**：改用 `GET /api/captain/available-players?roundId=round-X` 替代全局用户列表
