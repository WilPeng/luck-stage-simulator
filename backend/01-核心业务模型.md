## 一、核心实体定义

```typescript

  id: string

  createdAt: string
  updatedAt: string
}
```

### 1.2 Player（选手）

```typescript
interface Player {
  id: string
  name: string
  avatar: string | null
  role: 'player' | 'captain' | 'admin'
  status: PlayerStatus
  attributes: {
    vocal: number       // 声乐能力 1-100
    dance: number       // 舞蹈能力 1-100
```

### 1.3 Round（公演轮次）

```typescript
interface Round {
  id: string
  index: number          // 1 = 一公, 2 = 二公, 3 = 三公

}

type RoundStatus = 'pending' | 'active' | 'completed'
```

**注意：轮次是线性递增的，不能跳跃。**

### 1.4 RoundTeam（轮次队伍）

```typescript
interface RoundTeam {
  id: string
  roundId: string
  roundIndex: number     // 冗余存储，便于查询
  name: string           // 例如："乘风一队"
  captainId: string | null
  maxMembers: number
  locked: boolean        // 是否已锁定（锁定后不能再添加成员）

  createdAt: string

```

**⚠️ 重要：队伍只属于当前轮次。下一轮重新创建。**

## 二、阶段定义（绝对统一）

### 2.1 Stage 枚举

```typescript
enum Stage {
  PREPARATION = 'preparation',     // 预先准备
  CAPTAIN_VOTE = 'captain_vote',    // 队长选举
  TEAMING = 'teaming',              // 组队
  SONG_SELECT = 'song_select',      // 选歌
  TRAINING = 'training',            // 训练
  REHEARSAL = 'rehearsal',         // 彩排
  PERFORMANCE = 'performance',       // 公演
  ELIMINATION = 'elimination'       // 淘汰
}
```

### 2.2 阶段顺序（固定）

```
PREPARATION → CAPTAIN_VOTE → TEAMING → SONG_SELECT → TRAINING → REHEARSAL → PERFORMANCE → ELIMINATION
```

| 序号 | 阶段 | 说明 |
|-----|------|------|
| 1 | PREPARATION | 管理员配置本轮参数 |
| 2 | CAPTAIN_VOTE | 选手投票选队长 |
| 3 | TEAMING | 组队申请/邀请 |
| 4 | SONG_SELECT | 抢歌或指定歌曲 |
| 5 | TRAINING | 翻牌训练 |
| 6 | REHEARSAL | 彩排 |
| 7 | PERFORMANCE | 公演 |
| 8 | ELIMINATION | 淘汰 |

### 2.3 ⚠️ 禁止事项

- ❌ 禁止自行定义阶段字符串
- ❌ 禁止使用中文阶段名作为判断条件
- ❌ 禁止硬编码阶段顺序

## 三、当前阶段状态

### 3.1 系统唯一状态

```typescript
interface SystemState {
  currentRound: number    // 当前第几公演
  currentStage: Stage     // 当前阶段
}
```

**示例：**

```json
{
  "currentRound": 2,
  "currentStage": "TEAMING"
}
```

表示：**正在进行第二公演组队阶段**

### 3.2 状态计算规则

所有页面状态由后端统一计算返回：

```typescript
function calculateStatus(
  currentRound: number,
  currentStage: Stage,
  targetRound: number,
  targetStage: Stage
): StageStatus {

  const stageOrder = [
    Stage.PREPARATION,
    Stage.CAPTAIN_VOTE,
    Stage.TEAMING,
    Stage.SONG_SELECT,
    Stage.TRAINING,
    Stage.REHEARSAL,
    Stage.PERFORMANCE,
    Stage.ELIMINATION
  ]

  const currentIndex = stageOrder.indexOf(currentStage)
  const targetIndex = stageOrder.indexOf(targetStage)

  if (targetRound < currentRound) {
    return 'completed'
  }

  if (targetRound > currentRound) {
    return 'future'
  }

  // 当前轮次
  if (targetIndex < currentIndex) {
    return 'completed'
  }
  if (targetIndex === currentIndex) {
    return 'current'
  }
  return 'future'
}
```

### 3.3 StageStatus 类型

```typescript
type StageStatus = 'completed' | 'current' | 'future'

// completed: 已完成，可查看历史结果
// current: 当前阶段，可操作
// future: 未开始，禁用
```

## 四、业务规则

### 4.1 选手状态规则

| 状态 | 说明 | 可组队 | 可训练 | 可公演 |
|------|------|--------|--------|--------|
| active | 活跃 | ✓ | ✓ | ✓ |
| danger | 危险 | ✓ | ✓ | ✓ |
| eliminated | 淘汰 | ✗ | ✗ | ✗ |

### 4.2 队伍规则

1. 每轮开始时，管理员预先定义队伍数量和各队人数
2. 队伍在 ELIMINATION 阶段之前必须锁定
3. 锁定后的队伍不能再添加/移除成员
4. 每轮结束后，所有队伍清空，重新开始

### 4.3 歌曲规则

1. 每轮开始时，管理员从歌曲库选择本轮歌曲池
2. SONG_SELECT 阶段开始后，歌曲释放给用户抢歌
3. 每个队伍只能选择一首歌曲
4. 歌曲一旦分配，不能更改

### 4.4 训练规则

1. 每轮开始时，管理员设置每人训练次数
2. 每次训练消耗一次训练机会
3. 训练次数用完不能再训练
4. 训练效果是永久的（属性累加）

### 4.5 淘汰规则

1. 每轮公演结束后，根据排名确定淘汰名单
2. 管理员可以手动调整淘汰名单
3. 淘汰确认后，选手状态变为 eliminated
4. 淘汰选手不能再参加后续轮次

## 五、数据关系图

```
Season (赛季)
    │
    ├── Round (轮次) × N
    │       │
    │       ├── RoundTeam (队伍) × N
    │       │       │
    │       │       └── RoundTeamMember (队员) × N ─── Player
    │       │
    │       ├── RoundSong (本轮歌曲池) × N ─── Song (歌曲库)
    │       │
    │       ├── TeamSong (歌曲归属) × N
    │       │
    │       ├── TrainingRecord (训练记录) × N ─── Player
    │       │
    │       ├── RehearsalResult (彩排结果) × N
    │       │
    │       ├── TeamResult (队伍结果) × N
    │       │
    │       ├── PlayerResult (个人结果) × N
    │       │
    │       └── EliminationRecord (淘汰记录) × N
    │
    └── SystemState
            ├── currentRound
            └── currentStage
```
